# BoundaryML Positioning

## One-liner

BoundaryML is the project kickoff and governance layer for AI agents.

## Short Pitch

BoundaryML turns vague project ideas into agent-ready execution kits, so teams can hand work to Codex, Claude Code, GitHub Copilot, Cursor, or other coding agents with clear task boundaries, required context, output contracts, review gates, and acceptance criteria.

## Long Pitch

AI coding agents are becoming powerful execution engines, but most teams still give them vague tasks. BoundaryML solves the upstream problem: it prepares work before agents execute.

Instead of asking an agent to infer project boundaries, required context, expected outputs, review responsibilities, risk levels, and acceptance criteria, BoundaryML makes those boundaries explicit. The result is an agent-ready execution kit: workflow map, agent task list, prompt pack, review checklists, artifact contracts, responsibility map, and risk gates.

BoundaryML does not replace coding agents, issue trackers, or documentation tools. It creates the governance layer between a vague project idea and downstream AI-assisted execution.

## Positioning Against Alternatives

| Alternative | What it is | BoundaryML position |
| --- | --- | --- |
| Jira / Linear | Project and issue tracking systems. | BoundaryML prepares agent-ready task boundaries, prompts, review gates, and artifact contracts before or alongside issue creation. |
| Notion / Docs | Flexible documentation spaces. | BoundaryML turns loose docs into structured execution kits with validation and governance. |
| Miro / flowchart tools | Visual mapping and diagramming tools. | BoundaryML is not a generic diagramming tool; its workflow map is tied to agent execution boundaries, risk, ownership, and output contracts. |
| Codex / Claude Code / Copilot | AI coding agents and assistants that execute or assist with code work. | BoundaryML prepares the work before these agents execute. |
| LangGraph / workflow runtime | Runtime framework for stateful agent or application workflows. | BoundaryML is a planning and governance layer, not an execution runtime. |
| Dify / n8n / automation builders | AI app and automation builders. | BoundaryML defines project governance and agent-ready task packs rather than building automations or running production workflows. |
